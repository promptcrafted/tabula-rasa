**3. Add "Workflow Chaining" section (new, after Behavioral Guidelines):**

Content based on 06-RESEARCH.md Pattern 1 and Pattern 2. Include:

Pipeline Execution subsection:
1. Parse mode selection from user request (default: full pipeline research -> plan -> design -> oversee; partial runs like `research+plan`, `design+oversee` supported)
2. For each mode in pipeline: format context using handoff template, spawn mode agent, collect output (file path + recap), validate output (file was written, recap has content, no error signals), if validation fails retry once then stop pipeline and report what completed, generate milestone summary, identify risks from output
3. At design -> oversee transition: apply maker-checker loop (spawn overseer, if PASS proceed, if WARN note and proceed, if FAIL attempt 1 re-spawn designer with findings then re-oversee, if FAIL attempt 2 escalate to user). Maximum 2 maker-checker iterations.
4. After pipeline completion: comprehensive summary of all stages, identified risks across workflow, recommended next actions.

Context Handoff Templates subsection (CHST-05):
- Research -> Plan: key findings summary, technology decisions, constraints, pitfalls, file path to full report
- Plan -> Design: phase structure, acceptance criteria, dependency order, scope boundaries, file path to full plan
- Design -> Oversee: design deliverables list, file paths, acceptance criteria from plan, review focus areas
- Oversee -> Design (iteration): specific findings with severity, recommended fixes, which criteria failed, iteration count

Specify that handoffs are ephemeral conversation context passed via Agent tool prompt (not persisted files). Keep handoff summaries to 10-15 lines per stage to manage context budget.

- Read-only: never writes to `.planning/` or modifies GSD agents/hooks/settings
