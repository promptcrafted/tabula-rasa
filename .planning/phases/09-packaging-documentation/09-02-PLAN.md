  <name>Task 1: Create README.md</name>
  <files>README.md</files>
  <action>
    Create README.md at repo root. Tone: direct, slightly opinionated, developer-focused. Not a marketing page.

    **Structure (per CONTEXT.md decisions):**

    1. **Title + tagline**: "tabula-rasa" with one-line description ("Your AI chief of staff for Claude Code")

    2. **What is this** (1 paragraph): A chief of staff agent system for Claude Code that personalizes itself to you. Chains research, planning, design, and oversight into coherent workflows. Ships as a set of agent files that install into your `~/.claude/` directory. On first run, it interviews you and creates its own persona.

    3. **Quick install** (3-4 lines):
       ```
       git clone <repo-url>
       cd tabula-rasa
       bash install.sh
       ```
       Then: run `/setup` in Claude Code for first-time persona creation.

    4. **What you get** (5 modes, 1 line each):
       - `/signe-research` -- Multi-source investigation with confidence scoring
       - `/signe-plan` -- Goal decomposition with dependency mapping
       - `/signe-design` -- Architecture, UI/UX, agent, or product design
       - `/signe-oversee` -- Code review, quality gates, progress tracking
       - `/signe` -- Chain all modes into a single workflow

    5. **How it works** (brief): Flat orchestrator model -- main agent delegates to specialist subagents. Each mode has its own methodology. Memory persists across sessions.

    6. **Example interaction**: Show a single compelling example -- a `/signe-research` call or a full `/signe` pipeline. Keep it realistic but concise (10-15 lines of example). Show the kind of structured output the agent produces.

    7. **Setup walkthrough**: Brief description of the `/setup` experience -- agent asks about your work, generates its own name and personality, remembers you next time.

    8. **Documentation**: Link to `docs/guide.md` for full mode documentation.

    9. **License**: MIT (or whatever license file exists; if none, add MIT)

    **Do NOT include:** badges/shields, contributor guidelines, changelog, CI status.
    **Do NOT use emojis** in the README.

    **Sections (per CONTEXT.md decisions):**

    1. **Overview**: What tabula-rasa is, the flat orchestrator model, how modes relate to each other.

    2. **Installation**: Detailed install instructions including:

       - Clone and run install.sh

       - 1-sentence description
       - Invocation syntax (`/signe-research`, etc.)
